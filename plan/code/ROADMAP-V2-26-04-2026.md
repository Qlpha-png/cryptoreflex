## 🔥 P0 — À FAIRE CETTE SEMAINE (impact massif + effort faible)

### 1. Quick Win SEO — Hreflang multi-région (15 min)
**Source** : Audit SEO #1 (issue critique)
**Pourquoi** : Tu cibles FR/BE/CH/QC mais aucune balise hreflang → Google peut considérer les contenus dupliqués entre marchés.

**Code** : ajouter dans `app/layout.tsx` metadata :
```ts
alternates: {
  canonical: BRAND.url,
  languages: {
    "fr-FR": BRAND.url,
    "fr-BE": BRAND.url,
    "fr-CH": BRAND.url,
    "fr-CA": BRAND.url,
    "x-default": BRAND.url,
  },
},
```

### 2. Quick Win CRO — MobileStickyCTA sur `/avis/[slug]` (10 min)
**Source** : Audit CRO #6
**Pourquoi** : Pages avis de 200+ lignes sans CTA mobile fixe → utilisateurs mobiles abandonnent avant de cliquer.

**Code** : `MobileStickyCTA` est déjà importé dans `app/avis/[slug]/page.tsx`. Ajouter avant le footer :
```tsx
<MobileStickyCTA platformId={p.id} surface="avis-page" />
```

### 3. Quick Win Trust — "Updated April 26, 2026" sur tous les articles (15 min)
**Source** : Audit Trust #5 + Audit SEO #5
**Pourquoi** : E-E-A-T Google + signal "contenu maintenu" + pic CTR articles.

**Code** : Vérifier que `frontmatter.updatedAt` est affiché dans `components/MdxContent.tsx` ou template article. Si pas, ajouter `<time dateTime={updatedAt}>Mis à jour le {format(updatedAt, 'd MMMM yyyy')}</time>` en haut de chaque article.

### 4. Quick Win Trust — Author "Kevin Voisin" sur articles (5 min)

**Source** : Audits Content Gap + Competitor FR + INTL (3 agents convergent)

## 🚦 Méthodologie d'exécution

### Règles
1. **P0 d'abord** : tout ce qui est <30 min et impact >5% est prioritaire absolu
2. **A/B test obligatoire** : toute nouvelle UI majeure passe par `lib/abtest.ts`
3. **Mesure tout** : chaque expérience a sa metric Plausible custom event
4. **Iterate fast** : déploiement continu, pas de "release" mensuelle

### Outils déjà en place pour exécuter
- `lib/abtest.ts` : framework A/B léger, KV-backed
- `app/api/abtest/exposure` + `conversion` : tracking
- Plausible custom events
- Microsoft Clarity heatmaps
- KV Upstash pour analytics events
- Vercel CLI deploy `vercel --prod`

### Cadence recommandée
- **Daily 30 min** : monitoring (Plausible, Clarity, deploy logs, affiliate dashboards)
- **Weekly 4h** : 1 quick win SEO + 1 quick win CRO + 1 article publié
- **Monthly 8h** : 1 outil interactif majeur + 1 expérience A/B
- **Quarterly** : revue des KPIs + ajustement roadmap
