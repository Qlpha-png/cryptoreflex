## 3. Stratégie cookie / tracking / anti-fraude

### 3.1 Architecture UTM standardisée

Format unique imposé sur **tous** les liens d'affiliation sortants :

```
?utm_source=cryptoreflex
&utm_medium=affiliate
&utm_campaign=<programme>
&utm_content=<slug-page-source>
&utm_term=<position-cta-1|2|3>
```

Exemple : `https://www.bitget.com/fr/expref/abc123?utm_source=cryptoreflex&utm_medium=affiliate&utm_campaign=bitget&utm_content=guide-trading-debutant&utm_term=cta-2`

### 3.2 Couche de redirection interne (essentielle)

Mettre en place `cryptoreflex.fr/go/<programme>` (301 ou JS redirect avec délai 0).

**Bénéfices** :
- Permet de switcher de programme **sans toucher au contenu** (ex : si Bitget baisse la commission, switch vers OKX).
- Logs internes côté serveur (PostHog ou Plausible self-hosted) → on connaît les vrais clics même si le programme tracke mal.
- A/B testing au niveau de la redirection (2 codes affiliés différents sur la même URL pour comparer rev-share).
- Évite le "ad-blocker leak" qui casse les liens directs Impact/Awin.

### 3.3 Réseaux d'affiliation à exploiter

- **Impact.com** : Coinbase, Ledger, Koinly, Tangem, parfois Trezor/Revolut. UI propre, dashboard fiable.
- **Awin** : Trezor, Trade Republic, N26, Revolut. Frais d'activation 5 USD à régler, ne pas paniquer.
- **Direct (in-house)** : Binance, Bybit, OKX, Bitget, MEXC, Bitpanda, Coinhouse, Waltio, Bitstack. Toujours préférer le direct pour rev-share lifetime crypto.

### 3.4 Anti-fraude / protection des commissions

- **Disclosure visible** sur chaque page (mention légale "liens d'affiliation") = obligation DGCCRF + protection contre suspension.
- **Pas de cashback / no incentivized clicks** : interdit par la quasi-totalité des programmes (Binance, Bybit, OKX bannissent à vie).
- **Pas de PPC sur le brand** : ne jamais bidder "Binance" sur Google Ads → ban immédiat.
- **Server-side tracking** (CAPI Meta + GA4 Measurement Protocol) → résiste aux iOS 17+ et ad-blockers.
- **Backup attribution** : log first-click + last-click en parallèle pour disputes.
- **Reporting hebdo** : export CSV de chaque dashboard chaque lundi, archivage Drive (preuves en cas de litige paiement).

## 4. Optimisations de conversion

### 4.1 Placement des liens (par template)

| Type de page | Placement #1 (priorité haute) | Placement #2 | Placement #3 |
|--------------|-------------------------------|--------------|--------------|
| Avis exchange | CTA bouton sticky (mobile) au-dessus du fold + tableau "verdict" | Encadré "code promo exclusif" milieu d'article | CTA bouton final + footer comparatif |
| Comparatif | Tableau avec bouton "S'inscrire" sur chaque ligne | Top 3 cards en haut | CTA récap fin |
| Guide débutant | Encadré "Pour démarrer, on recommande X" après l'intro | Step-by-step avec screenshots cliquables | CTA secondaire en conclusion |
| Page fiscalité | Bouton "Déclarer avec Waltio -20 %" sticky | Comparatif logiciels avec CTA par ligne | — |
| Hardware wallet | Bouton "Acheter sur Ledger.com" + prix barré | Comparatif Ledger vs Trezor avec 2 CTA | Bandeau Black Friday saisonnier |

### 4.2 A/B tests à lancer dès semaine 2

1. **CTA wording** : "S'inscrire sur Binance" vs "Recevoir 100 USDT de bonus" vs "Profiter de -20 % de fees" → le bonus chiffré gagne en général de 30–50 %.
2. **Couleur bouton** : orange Binance-natif vs vert Cryptoreflex-brand → le natif convertit mieux (familiarité).
3. **Lien direct vs page intermédiaire** :
   - **Lien direct** (clic sortant immédiat) : meilleur CTR, attribution propre, recommandé pour les programmes lifetime (Binance, Bybit, Bitget).
   - **Page intermédiaire** ("/avis/binance" → CTA → exchange) : meilleur taux de FTD (le lecteur arrive "réchauffé"), recommandé pour les programmes CPA stricts (Coinbase, Trade Republic) où chaque FTD vaut 8–60 €.
   - **Règle** : si commission est rev-share lifetime → lien direct. Si CPA one-shot → page intermédiaire avec preuve sociale + bonus.
4. **Position bouton** : sticky scroll mobile vs in-content seulement → sticky = +20–40 % CTR sur mobile FR.
5. **Avant/après KYC anxiety** : ajouter "KYC en 5 min" sous le CTA pour rassurer.

### 4.3 Contenu d'aide à la conversion

- Code promo exclusif visible (négocier avec chaque AM : Bitget donne facilement -10 à -20 % fees).
- Captures écran du dashboard exchange (preuve d'usage personnel = +trust).
- Tutoriel pas-à-pas inscription (réduit l'abandon KYC).
- Mini-FAQ "Est-ce sûr ?", "C'est légal en France ?", "Et les impôts ?" sous chaque CTA.

## 6. Pitfalls à éviter

### 6.1 Programmes à risque (mauvais payeurs / UX cassée)

- **MEXC** : commissions élevées sur le papier, mais reporting buggé, support lent, retraits parfois retardés 30+ jours. À utiliser en complément, pas comme pilier.
- **OKX** : excellents taux mais a déjà unilatéralement réduit des paliers d'affiliés en 2024. Ne pas dépendre à >25 %.
- **Crypto.com** (volontairement non listé) : programme régulièrement fermé/réouvert, conditions opaques. À éviter.
- **eToro** : paiement CPA correct mais reverse rate très élevé (>30 %), KYC strict, expérience utilisateur dégradée en France post-MiCA.

### 6.2 Pièges contractuels

- **Clauses de "negative balance"** : si tes filleuls font perdre de l'argent à l'exchange (rare mais possible sur derivatives), commissions futures effacées. Bybit/Bitget : clause présente. Lire le TOS.
- **Clauses d'inactivité** : Binance désactive l'affiliate si <1 FTD/90 jours après la phase d'activation.
- **Clause de territorialité** : Trade Republic/N26 : commission uniquement sur leads géolocalisés FR. Vérifier que le tracking IP fonctionne.
- **Reverse / clawback long** : Coinbase peut clawback jusqu'à 60 j post-FTD. Provisionner.

### 6.3 Pièges techniques

- **Liens nofollow obligatoires** : tous les liens d'affiliation en `rel="sponsored nofollow"` (sinon pénalité Google + risque ban affilié).
- **Mention "liens affiliés"** obligatoire (DGCCRF, ARPP) — sinon procès consommateur potentiel et fermeture du programme côté annonceur.
- **Pas de cloaking agressif** : `bit.ly/binance-fr` ne marche plus, beaucoup de programmes interdisent les raccourcisseurs externes. Utiliser **uniquement** le redirect interne `/go/`.
- **iOS Safari ITP** : cookies tiers cassés. Backup avec tracking server-side (postback URL fournie par Bitget/Bybit/Binance).

### 6.4 Pièges éditoriaux

- **Ne pas survendre** : "10x garanti" → ban + amende AMF en France (publicité crypto encadrée par le statut PSAN/MiCA).
- **Toujours mentionner les risques** : "75 % des comptes perdent de l'argent" pour les derivatives = obligation MiCA depuis 2024.
- **Pas de classement biaisé** : si tu mets Bitget #1 alors qu'il paie le mieux, le préciser ("Pourquoi on recommande X"). Mieux pour la confiance et le SEO long terme.
- **Diversifier les sources de revenus** : viser <30 % du revenu sur un seul programme. Binance peut couper du jour au lendemain.

### 6.5 Pièges opérationnels

- **Ne pas laisser dormir les comptes** : connexion + 1 export par mois minimum (sinon désactivation silencieuse).
- **Centraliser les credentials** dans un password manager partagé (1Password / Bitwarden) avec MFA.
- **Tableau de bord unique** : agréger toutes les commissions hebdo dans un Google Sheet (ou Airtable). Sinon, perte de visibilité dès qu'on dépasse 10 programmes.
- **Anticiper la fiscalité** : revenus d'affiliation = BNC en France, à déclarer. Provisionner ~30 %.
